Launching into Computer Science

Learning progress throughout this module

Unit 1

In this unit, we began discussing computer science as a profession and the challenges this poses, such as handling data responsibly (in line with GDPR guidelines), securely and effectively. One of the key topics here is...ensuring organisational information security, normally by an IT policy. My research led me to understand that the main challenges here are managing stakeholder perceptions and expectations of any implemented policies and understanding the complexities and interdependenceies of organisational information.

I recommend the following two articles on this topic:

  • Samonas, S., Dhillon, G., & Almusharraf, A. (2020). Stakeholder perceptions of information security policy: Analyzing personal constructs. International Journal of Information Management, 50.
  • Diesch, R., Pfaff, M., & Krcmar, H. (2020). A comprehensive model of information security factors for decision-makers. Computers and Security, 92.

    Unit 3

    This week I used the ‘Learn with Google AI’ repository to explore a crash course in Machine Learning. I am interested in developing a sound understanding of this field and hope that it will also change the way I approach problems in future.

    Key points include:...

    "ML systems learn how to combine input to produce useful predictions on never-before-seen data."

    A Machine Learning model is trained by starting with an initial guess for the weights and bias and iteratively adjusting those guesses until learning the weights and bias with the lowest possible loss.

    A regression model predicts continuous values. For example, regression models make predictions that answer questions like the following:

    • What is the probability that a user will click on this ad?

    A classification model predicts discrete values. For example, classification models make predictions that answer questions like the following:

    • Is a given email message spam or not spam?

    I also began to explore the concept of bias in Machine Learning models, and how this might be built in to a model, depending on the context and parameters of the problem. Also, there are different ways to measure the accuracy of a model and it is possible to engineer the accuracy of a model, depending on the objective or on the resources available.

    Algorithms are increasingly being used to decide 'fair' outcomes. Is this ethical?

    I found the following recommended article very interesting

  • Can you make AI fairer than a judge?

    Unit 5

    In this unit we considered software, in terms of system software (e.g. operating systems) and application software. We were also introduced to the concept of ‘programming languages’ – Python, C++ etc. – and their different purposes as Assembler, Compiler of Interpreter....

    An operating system (OS) is the software controlling how a computer functions and operates, including storing and accessing files, executing programs and providing the necessary software to run certain applications.

    Windows is probably the best-known operating system around. It was developed in the 80s and is widely used on PCs but is not active in the mobile OS sector, where Android and iOS are the most commonly used. Windows is so popular because it can run a huge range of software and most programs are compatible with it. It is also necessary to buy a user license to use Windows. Windows also has gained a reputation for crashing or having service outage.

    Linux is a less commonly used OS and it is free of charge and open source, meaning that anyone can run or change the code to function differently for their use purpose. It is also supposed to be more stable and secure (i.e. less vulnerable to malicious attacks) than Windows. The user interface however is not as simple as windows is and some applications must be installed via the command line, which not everyone is familiar with.

    We also looked briefly at the process of developing software, which is often conducted according to SCRUM methodology. SCRUM is intended to be a very lean approach to the development of new software and applications, because it takes an iterative approach – i.e. development is structured in sprints, which work on one small piece of a larger project and aim to get that right before moving on to the next. In this way, improvements are always being made to the software and the work done is reviewed and improved by all members of the team, who each have a special role in the process.
