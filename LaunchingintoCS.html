Computing as a Profession

In this unit, we began discussing computer science as a profession and the challenges this poses, such as handling data responsibly (in line with GDPR guidelines), securely and effectively.     • Computing Infrastructure & Networks

    In this unit we focused on the networking capabilities of computer networks and the technology, topology, protocols and hardware involved.

    Some important definitions and concepts include:

    Network classifications such as PAN (Personal Area Network, for very short distance communications), LAN (Local Area Network, normally for several computers within one institution or office).

    Topology is the organization of the network – i.e. via a bus or wireless star network. The client/server model is an example of interprocess communication where the process machine (the client) makes a request of another process (the server) to complete the task at hand. The peer-to-peer (P2P)> model involves processes that require services from each other (not only from the server).

    Protocols are like a set of ‘rules’ which govern how communication activities are conducted. CSMA/CD (Carrier Sense, Multiple Access with Collision Detection) is a protocol used on a network of devices connected across a bus network where the machines will monitor all messages transmitted, keep the ones for itself and if required to transmit a message and a clash occurs, will try again later. This is not the case for machines connected via wireless networks (the standard being WiFi) which operate a CSMA/CA (Collision Avoidance) principle as the wireless nature of the communication means they cannot detect a collision.

    Hardware includes devices that facilitate the computer network. For example an Access Point is a device that creates a wireless local area network (WLAN), a repeater is a device which passes messages between buses, a bridge>passes messages between buses for a specific destination address and a switch connects several buses and also only sends messages to the specific destination. Routers are special computer devices which manage the connection between networks by forwarding messages to form an internet.

    In distributed computer systems, Cluster Computing is a distributed system where many computers work together to provide a service which would otherwise require a much larger machine, which is less costly than a supercomputer and leads to better availability of the service. Cloud Computing is a new trend in this area where a large pool of shared computers can be used by clients as and when they need it. Cloud services allow users to rent virtual computers without needing to know where the physical hardware is located. This allows for more flexibility in collaboration.

    The Internet is a collection of connected networks that today links millions of computers connected by PANs, LANs, MANs, WANs etc. An internet/ intranet is a network provided by a single authority to individuals. Each computer within an internet has an IP Address that basically identifies each computer within the system.

    A note on cyber security:

    The networking capabilities of computers can be very beneficial for improving quality of life and work, but are also vulnerable to cyber attacks. Viruses, worms, Trojan horses, spyware and phishing are all methods of compromising a computer network. Wireless communication enables transfer protocols to facilitate the remote control of a device, which is seen applied in Internet of Things (IoT) technology applications (e.g. cars, smarthomes and MedTech). However, without a full understanding of security risks involved in IoT technology, this could be cause great vulnerability to malicious attacks, especially as the application of wireless networking technology becomes more widespread across mutlitple networks. Cars are a good example of this but also healthcare is a risky application area. The ‘internet of medical things’ increases the interconnectivity of devices related to healthcare monitoring and patient data processing, but the increased interconnectivity of these devices and the broad range of devices/ networks on which data is stored create new paths for cyber attacks.

    Software as a Tool

    In this unit we considered software, in terms of system software (e.g. operating systems) and application software. We were also introduced to the concept of ‘programming languages’ – Python, C++ etc. – and their different purposes as Assembler, Compiler or Interpreter.

    An operating system (OS) is the software controlling how a computer functions and operates, including storing and accessing files, executing programs and providing the necessary software to run certain applications.

    Windows is probably the best-known operating system around. It was developed in the 80s and is widely used on PCs but is not active in the mobile OS sector, where Android and iOS are the most commonly used. Windows is so popular because it can run a huge range of software and most programs are compatible with it. It is also necessary to buy a user license to use Windows. Windows also has gained a reputation for crashing or having service outage.

    Linux is a less commonly used OS and it is free of charge and open source, meaning that anyone can run or change the code to function differently for their use purpose. It is also supposed to be more stable and secure (i.e. less vulnerable to malicious attacks) than Windows. The user interface however is not as simple as windows is and some applications must be installed via the command line, which not everyone is familiar with.

    We also looked briefly at the process of developing software, which is often conducted according to SCRUM methodology. SCRUM is intended to be a very lean approach to the development of new software and applications, because it takes an iterative approach – i.e. development is structured in sprints, which work on one small piece of a larger project and aim to get that right before moving on to the next. In this way, improvements are always being made to the software and the work done is reviewed and improved by all members of the team, who each have a special role in the process.

    If you make a new folder and look at the properties of this folder, the first thing I noticed was that the folder size and the size on disk is not the same. After some research, I realise that the size on disk represents the total number of bytes occupied on the hard drive – i.e. the number of bits reserved for the folder size, not the maximum size of a folder. This is the same with a file. From the properties, the user can also see the file or folder path, and control who has access to the data created (i.e. other users or clients in the user network).

    Notes on algorithms:

    An algorithm must have a well-established structure.

    The steps of an algorithm are sequenced – i.e. they are executed in the order they are given.

    An algorithm is abstract – one algorithm can be represented in many ways.

    A program is a representation of an algorithm.

    A process is the activity of executing a program.

    A primitive is a building block from which an algorithm can be constructed.

    Each primitive has its own syntax and semantics.

    A collection of primitives makes up a programming language.

    Pseudocode is a notational system used to express ideas formally during algorithm development.

    Iteration is the repetition of part of an algorithm until a condition is met or for a specified number of times.

    Indentation enhances the readability of a program.

    Recursive structures can be used to repeat a set of structures as a subtask itself.

    The development of a program consists of discovering the underlying algorithm and representing the algorithm as a program - it requires a method of problem-solving.

    Algorithms solve problems - first you must understand the problem before you can formulate the algorithm.

    Tips for algorithm development:

    Stepwise refinement - don't try to solve the whole task at once

    Use loops - this makes things more flexible instead of repeating code over and over

    Successful loop control - means initialize, test, modify

    Use lists - to organise information within an album (not variables)

    Follow principles of SCRUM - as an iterative approach to algorithm development

    Ask the right questions to identify the right data and process
