Why object-oriented programming?

This is a test pdf open here.

Object oriented programming is good at modelling reality. It is an incremental approach in which each component has a state and different ways of doing things, meaning it is much easier to debug, maintain and test. The modular approach means that deploying a new object within a program is easy – objects can be cut out and put it individually. It also is very in-keeping with the Agile approach to software development.

However, overdoing the object-oriented approach (i.e. including too many objects) makes it more difficult, especially if you work with developers who are used to working in a functional environment...

UML: Unified Modelling Language

According to Fowler (2013) UML is a graphical modelling language to help design and describe software, in the following ways:

1) UML as sketch – developers use it to help communicate aspects of a system

2) UML as blueprint – e.g. a designer builds the design of the program for a programmer to code

3) UML as programming language - developers draw UML diagrams that are compiled directly to executable code, and the UML becomes the source code.
