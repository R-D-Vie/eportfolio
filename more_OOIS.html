How does object-oriented programming work?

Object-oriented programming is a style of programming in which data and the operations that manipulate it are organized into classes and methods.

A class describes an object, defined by the programmer. Each class has a set of STATES (things that describe the object) and a set of BEHAVIOURS (things that the object can do). A class object is an object that contains information about a class. An instance is an object belonging to a class. A class attribute is a value associated with an object. Class attributes are defined inside a class definition but outside any method. An embedded object is an object stored as an attribute of another object. A method is a function that is defined inside a class definition and is invoked on instances of a particular class.

Classes can also be associated with each other. Inheritance is where a new class can be defined as a modified version of a previously defined class where the subclass inherits all the features of the parent class methods. The parent class is a class from which child class inherits. The child class/ sub class is a new class created by inheriting form an existing class. A child class has an ‘IS-A’ relationship with its parent class. A class may also have a ‘HAS-A’ relationship where instances (objects) of one class contain references to instances of another. If two classes are dependent they have a relationship where instances of one class use instances of the other.

Why do information systems fail?

Poor testing – lack of testing done before system goes into production. Ideally, testing should happen during development (i.e. unit-testing or behavioural-driven development)

Poorly defined requirements - not fully understanding the problem to be addressed or not fully appreciating the requirements before beginning to write code
