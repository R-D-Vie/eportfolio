UML

UML is not a programming language per se (there is some discussion around the usefulness of using UML to inform on programming), but tools exist to turn UML diagrams into code. In this module we used UML as a tool to design systems. Standard notation for UML has become less clear over the years (although there are generally accepted principles, in practice there is some variance) and it is generally used to devise a model (i.e. a diagram/ class diagram) that defines the concept at hand.

Fowler (2013) says that the UML development process can be either waterfall or iterative. Waterfall is a strict step by step process, which is now generally considered out of date due to it's inflexibility (although there may be some scenarios that still favour it): it involves 1) analysis of the problem; 2) system design; 3) coding; and 4) testing. An iterative approach is more common and means that you define each unit of functionality in separate iterations before moving on to the next.

Class diagrams are the most commonly used format in UML as they are an effective a way of designing and documenting an object-oriented system. A class diagram describes the types of objects and the various relationships that exist among them within a system. It also shows the properties and operations of a class and constraints which apply to the way the objects are connected.

Sequence diagrams are interaction diagrams which describe how groups of objects collaborate and how different classes communicate with each other to complete a task. They are not useful for precise definition of behaviors.

Click here to see an example of a sequence diagram for an appointment booking system in a doctor's surgery:

Appointment Sequence Diagram.

Activity diagrams are best used to describe business processes and work flows. The chart is intended to outline the essential sequencing to follow, and can depict processes which occur in parallel which need synchronisation. Activity diagrams a good way to represent behavioral logic (e.g. UX elements). Partitions are used to show which department or actor is doing what.
