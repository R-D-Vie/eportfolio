ReDOS and 'Evil Regex'

1. What is ReDOS and what part do ‘Evil Regex’ play?

ReDOS means Regular expression Denial of service, which is an attack that Regex expressions are vulnerable for exploitation by attackers, who take advantage of the fact that Regex may be so complex that they cause a program to work very slowly and subsequently to hand (Weidman, n.d.). Regex can become problematic is a naïve algorithm is used, as this type of algorithm will iterate through the input character by character to check for matches against the pattern. This is problematic as in this case, the increasing length of the input string will degrade the performance of the algorithm (a naïve regex algorithm usually has a time complexity of O(m*(n-m)) where n is the length of the input string and m is the length of the regex pattern - geeksforgeeks.com (2023)). An ‘Evil Regex’ is a Regex which includes a repetitive pattern and can get therefore stuck on certain input, meaning that it is vulnerable to be found by an attacker who will deliberately exploit the ‘evil’ regex component (Weidman, n.d.).

2. What are the common problems associated with the use of regex? How can these be mitigated?

As mentioned above, performance can be very slow, especially if the Regex involves backtracking where the algorithm will repeatedly return to positions to match a pattern in a different way (Weidman, n.d.). Another problem is that they are prone to error and when compiled will often will not be sufficiently error-checked as the compiler will only pick up syntax errors, of which Regex are generally free of (Larson & Kirk, 2018). Larson & Kirk (2018) propose a tool specifically to overcome the latter problem which generates test strings on the basis of a Regex and produces a list of accepted and rejected strings, allowing the user to debug the Regex effectively. However it may also be useful to break down complex Regex patterns into small parts and ensure that it is well commented and documented.

3. How and why could regex be used as part of a security solution?

Regex can be used for input validation/ sanitisation, access control and authorisation, redacting sensitive information (e.g. credit card numbers) or for analysisng suspicious patterns or activities in logs.
